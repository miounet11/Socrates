文档

最快的评估方式，是先安装，再按 preset 跑一遍真正的 brief。

判断 Socrates 是否适合你，最有效的路径不是看概念，而是实际安装、查看内置预设、 生成 request.yaml，然后跑一版 guided draft。

安装

uv tool install git+https://github.com/miounet11/Socrates.git
socrates init
export OPENAI_API_KEY=your-key

最短路径

socrates presets
socrates template blog_post --output request.yaml
socrates generate request.yaml --mode guided

CLI 参考

最值得先掌握的几个命令。

`socrates presets`

列出内置内容预设以及它们的推荐模式。

`socrates template PRESET`

根据内置 preset 生成一份 starter YAML request。

`socrates frame request.yaml`

只生成策略层 / rubric 层，便于直接检查 frame 是否靠谱。

`socrates generate request.yaml`

按 `auto`、`direct`、`guided` 或 `full` 模式生成内容。

`socrates review draft.md --request request.yaml`

对现有 markdown 草稿做一轮 review，检查发布前风险。

`socrates doctor`

检查 provider 配置是否可用，以及关键凭证是否到位。

`socrates site-generate`

发布长期资源页，并重建资源库索引页与 sitemap。

内置预设

这些 preset 对应的是常见而且真实的专业内容工作流。

Preset Mode Platform 用途
blog_post guided blog 适合带明确论点和更清晰结构的博客长文
linkedin_long_post guided linkedin 适合节奏更强、开头更有钩子的长帖
value_prop guided web 适合首页 hero、落地页和定位文案
industry_analysis full blog 适合结构更重、审稿要求更高的行业分析
content_calendar full multi-platform 适合跨渠道的编辑规划与系列内容设计
brand_narrative full web 适合高价值定位与品牌叙事内容

对象模型

产品表面刻意保持小而清晰。

ContentRequest

运行 brief。包含主题、受众、目标、平台、类型、约束和 tone notes。

ContentFrame

内容标准。包含痛点、说服点、可信度要求、语气规则、反模式和核心角度。

ContentOutline

内容结构。定义 opening hook、章节顺序、证据需求和 CTA 位置。

ContentDraft 与 ReviewReport

一个是对外文案，一个是内部审稿结果，两者分离能让交付更干净。
