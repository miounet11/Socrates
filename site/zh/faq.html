FAQ

把边界说清楚，才能让用户知道这是什么，也知道它不是什么。

简单说，Socrates 给专业团队一个更像“内容系统”的生成方式。它不会替你做研究， 也不会替你承担编辑判断，但它能让标准、结构和审稿流程更明确。

Socrates 是通用 agent 框架吗？

不是。它是一个面向专业内容系统的生成层。目标不是调度任意工具，而是把内容标准 在写正文之前先显式建立出来。

Socrates 会自动连接网络做事实核查吗？

当前版本不会。review 阶段主要处理发布性风险，比如重复、无支撑的确定性表达和 明显的 AI 味。事实核查仍然需要团队自行处理。

什么时候该用 direct、guided 或 full mode？

direct 适合摘要、改写等工具型任务；guided 适合博客、LinkedIn 和价值主张等 常规发布任务；full 适合品牌叙事、行业分析和内容日历等高价值内容。

为什么不直接把 prompt 写得更长一点？

因为再长的 prompt 也可能只是把标准藏在一堆说明里。Socrates 会把这些标准转成 可观察、可复用的类型化对象，便于团队验证和复盘。

当前支持哪些 provider？

当前版本默认提供 OpenAI-compatible chat completions 适配层。provider 边界 设计得比较窄，目的是方便后续继续接入新 provider。

这个网站本身适合直接对外访问吗？

现在可以。站点已经提供英文和简体中文双语版本，并且已经部署在带 HTTPS 的 Nginx 静态站点环境中，便于国内外用户直接访问。
